**Tool definition** using `@langchain/core/tools` `tool()` helper:

```ts
const transcriptTool = tool(
  async ({ meetingId }: { meetingId: string }) => {
    const chunks = getTranscriptChunks(meetingId);
    if (!chunks || chunks.length === 0) return 'No transcript available for this meeting.';
    return chunks.map(c => c.speaker ? `${c.speaker}: ${c.text}` : c.text).join('\n');
  },
  {
    name: 'fetch_transcript',
    description: 'Fetch the full transcript for a specific meeting by ID. Use this when the user asks about specific things that were said in an interview.',
    schema: z.object({ meetingId: z.string().describe('The meeting ID to fetch transcript for') }),
  }
);
```

**Model binding**: Use `ChatOpenAI` from `@langchain/openai` with `model: 'gpt-4o-mini'`, `apiKey: process.env.OPENAI_API_KEY`, `streaming: true`. Call `.bindTools([transcriptTool])` on the model.

**Export `streamAgentResponse`** with signature:
```ts
export async function streamAgentResponse(
  message: string,
  context: string,
  resume: string,
  history: ChatMessage[],
  meetingId: string | undefined,
  onToolCall: (toolName: string) => void,
  onToken: (token: string) => void
): Promise<void>
```

**Agent loop** (manual tool loop — do NOT use AgentExecutor, it doesn't support token-level streaming cleanly):
1. Build messages array: SystemMessage (context + resume injected same as openai.ts), history messages (HumanMessage/AIMessage), new HumanMessage.
2. Call `model.stream(messages)` — iterate chunks, accumulate content and any tool_calls from the stream.
3. If the streamed response contains tool_calls (check `chunk.additional_kwargs?.tool_calls` or aggregate from AIMessageChunk): call `onToolCall('fetch_transcript')` once, execute the tool, append `AIMessage` with tool_calls and `ToolMessage` with result to messages, then call `model.stream(messages)` again for the final answer — stream tokens via `onToken`.
4. If no tool_calls: stream tokens directly via `onToken` as chunks arrive.

**Tool call detection**: LangChain streams tool calls incrementally in `AIMessageChunk.tool_call_chunks`. Accumulate them into a single `AIMessage` tool call. Use `concat()` from `@langchain/core/messages` to aggregate AIMessageChunks if needed.

**Error handling**: Wrap tool execution in try/catch — if tool throws, log the error and continue with final answer using original RAG context (do not re-throw). The onToken fallback still produces an answer.

Note: Do NOT import from `retrieve.ts` — the RAG context is already computed in route.ts and passed as the `context` parameter. The transcript tool fetches the raw stored chunks directly from vectorStore for full fidelity.

**Updated handler**:
```ts
import { streamAgentResponse } from '@/lib/chatbot/agent';
// Remove: import { streamChatResponse } from '@/lib/chatbot/openai'

export async function POST(req: NextRequest) {
  try {
    const { message, resume, history = [], meetingId } = await req.json();
    const context = await getRelevantDocs(message, meetingId as string | undefined);
    const encoder = new TextEncoder();

    const stream = new ReadableStream({
      async start(controller) {
        try {
          await streamAgentResponse(
            message,
            context,
            resume || '',
            history as ChatMessage[],
            meetingId as string | undefined,
            (toolName) => {
              controller.enqueue(
                encoder.encode(`data: ${JSON.stringify({ toolCall: toolName })}\n\n`)
              );
            },
            (token) => {
              controller.enqueue(
                encoder.encode(`data: ${JSON.stringify({ answer: token })}\n\n`)
              );
            }
          );
          controller.close();
        } catch (error: unknown) {
          const msg = error instanceof Error ? error.message : 'Failed to get response';
          controller.enqueue(
            encoder.encode(`data: ${JSON.stringify({ error: msg })}\n\n`)
          );
          controller.close();
        }
      },
    });

    return new Response(stream, {
      headers: {
        'Content-Type': 'text/event-stream',
        'Cache-Control': 'no-cache',
        'Connection': 'keep-alive',
      },
    });
  } catch {
    return new Response(JSON.stringify({ error: 'Internal server error' }), {
      status: 500,
      headers: { 'Content-Type': 'application/json' },
    });
  }
}
```
